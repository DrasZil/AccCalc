AccCalc Progress Summary

This document summarizes the major app changes completed so far across the accounting companion project, including features, Smart Solver upgrades, shell UX, deployment handling, PWA behavior, and offline user experience improvements.

Version 1.0.0 • Generated April 4, 2026 • Static-site PWA

1. Core product expansion

  • Expanded the app from a small calculator set into a larger multi-category toolkit covering finance, business, accounting, smart routing, settings, and persistent user history.
  • Upgraded the basic calculator to behave more like a real calculator with memory controls, better expression handling, parentheses, CE/AC, unary actions, keyboard support, and reusable recent calculations.
  • Added a true settings area and later converted it into a toggleable settings drawer so users can open and close settings without losing context.
  • Added a dedicated history area so users can reopen previous activity and saved prompts directly from the app.

2. Smart Solver improvements

  • Expanded Smart Solver from a limited matcher into a broader natural-language interpreter that works across finance, business, and accounting pages.
  • Improved vocabulary mapping so casual terms, student wording, and more professional accounting language can point to the same concepts.
  • Added dynamic input adaptation so the visible form changes based on the detected calculator and extracted accounting fields.
  • Added better calculator switching, extracted value application, detected currency handling, and stronger prompt-to-route behavior.
  • Started saving Smart Solver prompts into offline activity history for later review.

3. Accounting coverage added

  • Added partnership-related tools including profit sharing, admission under bonus method, admission under goodwill method, and salary-and-interest distribution.
  • Added depreciation and inventory analysis coverage including straight-line, declining balance, units-of-production, FIFO, weighted average, and gross profit method support.
  • Added manufacturing and cost-accounting topics including cost of goods manufactured, prime cost and conversion cost, materials price variance, and labor rate variance.
  • Added accounting analysis and ratio tools including current ratio, quick ratio, receivables turnover, inventory turnover, accounts payable turnover, debt-to-equity, debt ratio, return on assets, return on equity, asset turnover, times interest earned, EPS, and book value per share.
  • Added statement-analysis tools such as horizontal analysis and vertical analysis, along with Philippine VAT and other practical classroom topics.

4. Explanations and learning support

  • Enhanced the result/explanation pattern so calculators can show formulas, worked steps, glossary-style definitions of accounting terms, and plain-language interpretations.
  • Applied this explanation structure across many accounting, business, and finance pages to make the app more useful for both students and practitioners.
  • Improved readability by keeping meaning and interpretation close to the actual computed outputs instead of leaving the user with numbers only.

5. Navigation and interface improvements

  • Reworked the sidebar into a collapsed-first structure to reduce clutter and improve category hierarchy.
  • Moved About and Feedback into Settings and later added a history route and better top-level app organization.
  • Added icon-based shell controls, improved spacing and hierarchy, reduced header height on smaller screens, and made the shell more cohesive across desktop and mobile.
  • Added new-feature indicators so users can see what is newly added and have that marker disappear after opening the item once.
  • Added premium but restrained motion, startup visuals, and cleaner app-surface transitions without turning the UI into heavy or laggy animation.

6. Offline history, recommendations, and reminders

  • Added a client-side activity store backed by local storage so route visits, saved prompts, calculator outputs, and usage patterns persist after the app is closed.
  • Added personalized recommendations driven by user activity and category usage.
  • Added a feedback reminder flow that appears only after meaningful usage and can be postponed or disabled permanently.
  • Added device-local history controls in Settings, including the ability to clear stored activity.

7. Currency and regional handling

  • Introduced app-wide currency settings with PHP as the default but support for many common currencies.
  • Made Smart Solver detect prompt currency markers such as peso words, PHP, the peso sign, USD, and other common currency symbols.
  • Updated formatting helpers so result cards and explanations follow the selected or detected display currency.

8. PWA and deployment handling

  • Improved service worker behavior so production installs update more aggressively and dev mode unregisters workers to avoid stale localhost behavior.
  • Added update-takeover behavior, update notifications after refresh, and stronger cache version naming under semantic versioning.
  • Changed the app to use hash-based routing for static deployment so deep pages work better on hosts without server-side rewrites.
  • Added path-rewrite handling and app-shell fallback behavior to reduce refresh and page-not-found problems on deployed web and installed PWA use.
  • Improved manifest and shell metadata for more reliable install and update behavior.

9. Stability and error handling

  • Added an app-wide render error boundary so users do not just see a blank screen when a render failure happens.
  • Added user-facing notice handling for runtime issues and background promise failures.
  • Added startup buffering and safer launch behavior, especially for installed PWA use.
  • Cleaned several shell and navigation issues that previously caused blank or inconsistent rendering.

10. Current verification status

  • TypeScript compilation currently passes with npx tsc -b.
  • The production build in this machine still fails because Vite/Tailwind cannot load the native @tailwindcss/oxide-win32-x64-msvc dependency and also hits a local spawn EPERM issue. That is an environment-specific tooling problem rather than a TypeScript app error.
  • The app code has been structured for static deployment and installed PWA use, but the final production bundle still needs a machine where the Vite/Tailwind native dependency can load normally.
