AccCalc v12.1.0

Focus-First Shell Stabilization

Version 12.1.0 hardens the responsive shell, fixes the mobile menu bug at the root, and keeps major product surfaces more task-first.

Main Changes

  • Fixed the mobile navigation and partial-screen overlay bug with viewport portals and shared live-height handling.
  • Added shared body scroll locking for mobile navigation, search, settings, and page menus.
  • Simplified the shell context rail so global wayfinding stays helpful without dominating the page.
  • Reorganized the homepage around best-next actions, resume flow, and delayed discovery.
  • Moved lesson context and settings navigation into calmer progressive-disclosure patterns.

Focus-First Rules

Visible now

Inputs, main actions, main outputs, essential warnings, and the current workflow state.

Discoverable later

Related tools, deeper study notes, route context, support metadata, and broader browsing.

Mobile Shell Rules

  • Render temporary overlays at the viewport layer, not inside clipping layout containers.
  • Lock background scroll when a temporary surface is open.
  • Size menus, sheets, and drawers from the live viewport height.
  • Do not let bottom navigation compete with active overlays.

Shared Surfaces

  • AppLayout.tsx now owns a more reliable overlay and viewport model.
  • ViewportPortal.tsx standardizes rendering shell overlays against document.body.
  • useBodyScrollLock.ts coordinates background scrolling for temporary surfaces.
  • HomePage.tsx now emphasizes next-step decisions instead of equal-weight browsing.
  • StudyLessonLayout.tsx keeps lesson context secondary to the reading surface.
  • SettingsContent.tsx groups controls around category jumps and progressive disclosure.
