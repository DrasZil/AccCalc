AccCalc v10.5.0 Documentation

AccCalc v10.5.0 is the Student Reliability + Workpaper 2.0 release. It hardens the broad calculator, study, OCR, Smart Solver, and workpaper ecosystem so students can use the app with more confidence even if future maintenance is limited.

What Changed

  • Workpaper Studio now shows workbook health cues for formula errors, empty sheets, and row/column limit pressure.
  • Workpaper sheets are bounded at creation, import, transfer, and saved-state sanitation boundaries.
  • Oversized imports are intentionally truncated to the supported student-workpaper size with an explanatory note.
  • Smart Solver pauses before routing when confidence is low or extracted values require review.
  • Scan & Check opens review before routing when OCR confidence, parse confidence, flags, or structured fields are risky.
  • The spreadsheet import/export dependency is loaded on demand instead of being bundled into the default Workpaper Studio route.

Workpaper 2.0 Architecture

WorkpaperStudioPage.tsx remains the editor shell. It combines selected-cell editing, formula preview, templates, transfers, details, import/export actions, and the new health strip.

workpaperUtils.ts now owns the survivability caps for rows and columns. workpaperStore.ts sanitizes local workbooks before rendering. workpaperFile.ts dynamically imports xlsx only when import/export is requested.

The workpaper cap is intentional. AccCalc workpapers are browser-based student support sheets, not a full replacement for a desktop spreadsheet application.

Student-Trust Routing

Smart Solver and Scan & Check remain fast, but uncertain automation now slows down. Low-confidence route matches, flagged OCR values, weak parse confidence, or review-needed structured fields open a correction/review step before routing into a calculator.

This is a safety behavior, not a bug. The user can still continue after checking the risky values.

Calculation Assumptions

  • Confidence intervals explain whether z or t logic is being used.
  • Capital rationing exact search assumes independent indivisible projects and is capped at 20 positive-NPV projects.
  • Tax, audit, and classroom-model helpers should keep assumptions visible in page copy and result interpretation.

Performance Notes

The Workpaper Studio route is substantially lighter after moving xlsx into a separate on-demand chunk. Future optimization candidates include the large app catalog, formula solve definitions, Smart Solver route surfaces, and MathText/KaTeX rendering surfaces.

Validation

The final v10.5.0 handoff state is validated with npm test, npm run build, and a bounded npm run dev probe.
