AccCalc v6.0.0 Documentation

Release date: 2026-04-21

This documentation package is written for future maintainers, instructors, and student developers who may need to understand AccCalc with low prior context. It focuses on architecture, responsibilities, extension points, and the 6.0.0 release changes.

Project Overview

AccCalc is an accounting-learning and problem-solving web app built with React, TypeScript, and Vite. The product combines formula calculators, study content, OCR-assisted route suggestions, and workbook-style workpapers. The design goal is not only computation, but guided understanding and long-term maintainability.

Architecture

AccCalc is organized around a route shell, shared UI primitives, shared calculation logic, feature pages, and regression tests. The most important maintainability decision in the app is that calculation rules, reverse-solve logic, learning content, and route discovery are shared rather than page-local.

Route shell: src/App.tsx

Catalog and search: src/utils/appCatalog.ts, src/utils/appSearch.ts

Math and solve logic: src/utils/calculatorMath.ts, src/utils/formulaSolveDefinitions.ts

Study support: src/utils/formulaStudyContent.tsx, src/features/study/*

Smart Solver and OCR: src/features/smart/*, src/features/scan-check/*

Workpapers: src/features/workpapers/*

Important Folders

src/components

Shared shells, cards, grids, formula panels, interpretation blocks, and page layout primitives.

src/features/accounting

FAR and foundational accounting calculators such as reconciliation, receivables, inventory, and adjustment pages.

src/features/business

Managerial and cost pages including CVP, budgeting, performance, and planning tools.

src/features/afar

AFAR and consolidation helpers for combinations, intercompany profit, FX translation, construction, and new equity-method and PPE-transfer coverage.

src/features/smart

Natural-language routing and extracted-input logic for Smart Solver.

src/features/workpapers

Workbook state, spreadsheet formulas, templates, import/export, and studio UI.

Shared File Responsibilities

calculatorMath.ts: shared pure computations. This is the first place to look when a numeric answer is wrong.

formulaSolveDefinitions.ts: solve targets, validation, reverse solving, interpretation, warnings, and explanation steps.

formulaStudyContent.tsx: calculator-linked learning panels and concept reinforcement.

taxConfig.ts: editable Philippine tax defaults and visible assumption notes for time-sensitive rules.

appCatalog.ts: labels, aliases, tags, grouping, offline notes, and route discoverability.

workpaperTemplates.ts: aligned starter schedules for classroom workflows.

How Modules Connect

  1. A route in App.tsx loads a page.
  2. The page usually wraps FormulaSolveWorkspace or another shared shell.
  3. The solve definition chooses the visible inputs and target.
  4. The definition calls shared helper math.
  5. The result is rendered as cards, steps, and notes.
  6. Study support, Smart Solver, OCR routing, and workpaper templates point back to the same route.

Smart Solver Flow

The Smart Solver reads natural-language prompts, extracts numbers by alias, ranks calculators, and can suggest a solve target. This lets a user type something like “find the required labor hours” and land on the correct calculator with the correct target preselected.

OCR Routing Flow

OCR route suggestions do not compute answers directly. They classify scanned wording and recommend the best tool. The 6.0.0 release adds route hints for labor budgets, overhead budgets, budgeted income statements, note discounting, equity method, and intercompany PPE transfers.

Workpaper System

Workpaper Studio is the workbook layer of AccCalc. It supports multi-sheet workbooks, local autosave, formulas, formatting, import/export, and subject-aligned templates. The 6.0.0 templates were added specifically so new curricular breadth is usable in real assignments.

Testing

Shared regression coverage lives primarily in tests/calculatorMath.test.ts. This file checks calculations, search results, solve targets, workpaper helpers, and 6.0.0 additions. Always run npm test and npm run build before closing work.

How To Extend Safely

  • Add shared math first.
  • Add solve definitions next.
  • Then add the page, route, and catalog metadata.
  • Add Smart Solver and OCR coverage when discoverability matters.
  • Add a workpaper template if the workflow benefits from structured schedules.
  • Finish with tests and docs.

Known Limitations

  • Large production chunks remain in some areas, especially Workpaper Studio and the main app shell.
  • Tax pages are educational aids and still require human review when rules change.
  • Some advanced accounting areas remain intentionally simplified to keep the app stable and teachable.
