AccCalc v8.1.0

This handoff package documents the final 8.1.0 state of AccCalc after the focused follow-up release that fixed Windows local development, cleaned up the theme settings experience, and added transfer-pricing support to the managerial-accounting catalog.

Release Summary

  • Fixed the Vite config startup regression affecting npm run dev on Windows.
  • Moved runtime Vite loading to vite.config.mjs with native loading.
  • Made package metadata parsing BOM-safe inside the config path.
  • Fixed theme-family picker wrapping and stale appearance summary behavior.
  • Added Transfer Pricing Support with route, discovery, study, and workpaper wiring.

Vite And Local Development Fix

The highest-priority 8.1.0 fix was restoring local development on Windows.

Root cause: the previous startup path allowed Vite config loading to go through a fragile bundled path that choked on Tailwind's native Windows dependency. The fix was to move runtime loading to vite.config.mjs and run it with --configLoader native.

The config now reads package.json with BOM-safe parsing before injecting the runtime version string. vite.config.d.ts is only a TypeScript-side stub to keep config tooling explicit without reintroducing the old runtime problem.

Theme System And Settings Cleanup

Theme state has two axes: mode (system, light, dark) and family (classic, ocean, slate, rose, blossom, lavender, sunset, emerald).

  • Theme state is restored before paint through index.html.
  • The shell applies data-theme and data-theme-family for CSS-token theming.
  • 8.1.0 fixes the larger-screen theme-card overlap by using a more forgiving grid and wrapped swatches.
  • The quick appearance summary now updates when the theme family changes.
