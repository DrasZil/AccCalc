AccCalc v7.0.0 Documentation

Release date: 2026-04-21

AccCalc is a curriculum-aligned accounting companion built with React and TypeScript. It combines formula calculators, reviewer workspaces, OCR-assisted routing, Smart Solver discovery, and workbook-style workpapers so students and maintainers can move from problem identification to solution support in one system.

Project Overview

The app is organized around accounting and business curriculum families instead of purely technical layers. That choice makes navigation easier for users and also makes handoff easier for future developers who need to understand why a page exists and what academic area it supports.

Architecture Overview

Routing

src/App.tsx lazy-loads the route pages and keeps feature families grouped.

Shared UI

src/components provides the calculator shells, result cards, disclosure panels, and page layouts.

Shared Math

src/utils/calculatorMath.ts holds reusable business and accounting computations.

Solve Definitions

src/utils/formulaSolveDefinitions.ts keeps input fields, solve targets, steps, and interpretation aligned with the math.

Discovery

Search, Smart Solver, and OCR routing all depend on the shared route catalog and naming discipline.

Tests

tests/calculatorMath.test.ts protects shared helpers, search, workpaper utilities, and solve-target behavior.

Routing and Page Structure

The main route registry is src/App.tsx. Important route families include accounting, far, afar, business, operations, tax, audit, ais, rfbt, strategic, study, smart, and workpapers.

Major Shared Files

  • src/utils/appCatalog.ts: route metadata, aliases, taxonomy, and discovery tags
  • src/utils/appSearch.ts: route-search scoring and the precomputed search index
  • src/utils/calculatorMath.ts: formula and workspace computations
  • src/utils/formulaStudyContent.tsx: calculator-linked learning support
  • src/utils/taxConfig.ts: centralized editable tax assumptions
  • src/features/smart/smartSolver.engine.ts: extraction aliases and calculator configs
  • src/features/scan-check/services/ocr/ocrRouting.ts: OCR route suggestion rules
  • src/features/workpapers/workpaperTemplates.ts: starter workbook templates

How A Calculator Works

  1. A route page loads from src/App.tsx.
  2. The page passes a solve definition into FormulaSolveWorkspace.
  3. The solve definition validates inputs and calls a helper in calculatorMath.ts.
  4. The same solve definition returns steps, supporting results, assumptions, warnings, and interpretation.
  5. Study and related-route panels help the user continue to the next useful page.

Workpaper System

The workbook and workpaper layer lives under src/features/workpapers. It supports starter templates, workbook editing, formulas, formatting, import-export flows, and calculator-adjacent schedules. Version 7.0.0 extended this system with templates for special-order analysis, make-or-buy analysis, sell-or-process-further, constrained-resource product mix, budget variance analysis, and moving-average forecasting.

Search, Smart Solver, and OCR

appCatalog.ts is the shared backbone for route identity. Search uses appSearch.ts, which now precomputes a search index to reduce repeated candidate-building work. Smart Solver uses field metadata and calculator configs to interpret prompts, while OCR routing maps extracted text patterns to likely destinations.

Testing and Validation

The regression suite covers core math helpers, route search, solve-target behavior, and workpaper utilities. The 7.0.0 release adds direct tests for new managerial decision and forecasting helpers plus the new solve definitions and search coverage.

How To Safely Add A New Calculator

  1. Add the shared helper in calculatorMath.ts.
  2. Add a solve definition in formulaSolveDefinitions.ts if the calculator fits the shared workspace.
  3. Create the route page under the correct feature folder.
  4. Register the route in App.tsx.
  5. Add metadata and aliases in appCatalog.ts.
  6. Add Smart Solver and OCR support where discoverability matters.
  7. Add study support and workpaper templates where the workflow benefits.
  8. Add tests.

Known Limitations

  • Some heavy bundles still produce Vite chunk-size warnings.
  • The constrained-resource screen evaluates one product at a time rather than solving full multi-product optimization.
  • Tax tools remain educational helpers rather than filing systems.
